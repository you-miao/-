校园闲置物交易平台 - 项目需求文档
一、项目概述
本项目是基于 Spring Boot + Vue 前后端分离架构的校园闲置物交易平台，旨在为在校学生提供闲置物品的交换和售卖服务，实现闲置资源的高效流转，核心包含用户管理、商品管理、交易 / 交换流程、评论互动、后台管理等核心模块，满足学生日常闲置物处置需求。
二、核心功能需求
1. 用户管理模块
表格
功能点	详细需求
用户注册	1. 支持手机号 / 学号 + 验证码 / 密码注册
2. 填写基础信息（姓名、性别、校区、联系方式）
3. 校验信息唯一性（学号 / 手机号不可重复）
4. 注册成功后自动登录并跳转首页
用户登录	1. 支持手机号 / 学号 + 密码登录，或验证码快捷登录
2. 记住密码、自动登录功能
3. 登录失败提示（账号不存在 / 密码错误）
4. 登录后生成 JWT 令牌，维护登录状态
个人中心	1. 查看 / 修改个人信息（头像、联系方式、收货地址等）
2. 查看我的发布（交换物 / 售卖物）、我的订单 / 交换记录
3. 查看我的收藏、我的评论
4. 退出登录、修改密码
权限控制	1. 区分普通用户、管理员两种角色
2. 普通用户仅可操作自身资源，管理员可管理全平台数据
3. 未登录用户仅可浏览商品，不可发布 / 交易 / 评论
2. 商品管理模块
表格
功能点	详细需求
商品发布	1. 支持发布 “交换物” 或 “售卖物” 两类商品
2. 填写商品信息（名称、分类、描述、价格 / 交换条件、校区、联系方式）
3. 上传多张商品图片（限制格式 / 大小，支持预览 / 删除）
4. 发布后需管理员审核，审核通过后上架展示
商品列表	1. 展示商品封面、名称、价格 / 交换条件、发布人、发布时间
2. 支持按分类、校区、价格区间、发布时间筛选
3. 支持关键词搜索（匹配商品名称 / 描述）
4. 分页加载，下拉刷新 / 上拉加载更多
商品详情	1. 展示商品全部信息（图片轮播、详情描述、发布人信息）
2. 显示商品状态（待审核 / 已上架 / 已交换 / 已售出 / 已下架）
3. 提供 “收藏”“申请交换”“购买”“评论” 按钮
4. 关联推荐同类商品
商品管理	1. 发布者可编辑 / 下架自身商品
2. 管理员可审核 / 驳回 / 下架任意商品
3. 支持按状态（待审核 / 已上架 / 已下架）筛选自身发布的商品
3. 交易 / 交换流程模块
表格
功能点	详细需求
交换申请	1. 普通用户可向交换物发布者发起交换申请
2. 填写交换方案（拟交换的物品、沟通说明）
3. 发布者可同意 / 拒绝申请，双方可在线沟通
交换确认	1. 申请同意后，生成交换记录，记录交换双方、物品、约定地点 / 时间
2. 交换完成后，双方确认，商品状态改为 “已交换”
商品购买	1. 售卖物支持下单购买，填写收货信息 / 自提方式
2. 生成订单，记录订单号、商品、价格、状态（待付款 / 待收货 / 已完成 / 已取消）
3. 支持订单状态跟踪，收货后确认完成
4. 评论模块
表格
功能点	详细需求
评论发布	1. 已登录用户可对商品发布评论（文字 + 评分）
2. 评论需审核后展示，支持图片上传
3. 发布者可回复评论
评论展示	1. 商品详情页展示评论列表（按时间 / 评分排序）
2. 显示评论人、评论时间、评分、内容
3. 支持评论点赞
评论管理	1. 发布者可删除自身评论，管理员可删除任意评论
2. 支持举报违规评论，管理员审核后处理
5. 后台管理模块
表格
功能点	详细需求
商品审核	1. 查看待审核商品列表，支持通过 / 驳回（填写驳回理由）
2. 查看已审核商品，支持批量下架 / 删除
用户管理	1. 查看全平台用户列表，支持按学号 / 手机号搜索
2. 禁用 / 启用用户账号，查看用户发布记录
数据统计	1. 统计平台商品总数、交易 / 交换次数、活跃用户数
2. 按时间（日 / 周 / 月）展示数据趋势，生成可视化图表
3. 统计热门商品分类、高频交易校区
4. 易用性需求
表格
维度	要求
操作体验	1. 页面布局清晰，核心功能按钮显眼
2. 表单填写有实时校验，错误提示明确
3. 关键操作（发布商品、确认交易）有二次确认
容错性	1. 网络异常时提示用户，支持重新加载
2. 表单提交失败时保留已填写内容，不丢失数据
四、技术栈要求
表格
端别	技术选型
后端	1. 框架：Spring Boot 2.x/3.x
2. 数据库：MySQL 8.0
3. 安全：Spring Security + JWT
4. ORM：MyBatis-Plus
5. 图片存储：本地存储 / 阿里云 OSS
6. 接口文档：Swagger/knife4j
前端	1. 框架：Vue 3 + Vite
2. 路由：Vue Router 4
3. 状态管理：Pinia
4. UI 组件：Element Plus
5. 网络请求：Axios
6. 样式：SCSS/CSS3
总结
核心功能覆盖用户、商品、交易 / 交换、评论、后台管理五大模块，明确了各模块的具体操作流程和权限边界；
非功能需求从性能、安全、兼容性、易用性四个维度制定了可落地的指标，适配校园场景的使用规模；
技术栈选择主流的 Spring Boot+Vue 生态，兼顾开发效率和系统稳定性，符合前后端分离的架构设计。
如果需要针对某个模块（比如 “商品审核流程”“订单状态流转”）细化需求，或者补充原型设计、测试用例等内容，我可以进一步完善。
